# E2B Web Demo â€” Data Analysis Agent

Interactive WebUI demo: browser -> `agent-bundle dev` -> Agent -> E2B sandbox -> pandas/matplotlib analysis -> results + charts.

The agent analyzes data in a cloud sandbox using pandas, matplotlib, and numpy. It writes Python scripts, executes them, and returns statistical summaries and charts.

## Prerequisites

- Node.js >= 20 and pnpm
- API secrets:
  - `E2B_API_KEY` (for E2B sandbox + template build)
  - `OPENROUTER_API_KEY` (for OpenRouter model)

Run all commands from repository root.

## Quick start

One command handles everything â€” E2B API validation, bundle build, template build, and WebUI startup:

```bash
E2B_API_KEY=... OPENROUTER_API_KEY=... pnpm demo:web-e2b
```

The script is idempotent: on repeat runs, already-built templates are reused automatically.

Once running, open your browser:

`http://localhost:3000` (or the auto-detected worktree port)

### LLM provider

The demo ships with `provider: openrouter` / `model: google/gemini-2.5-flash` in `agent-bundle.yaml`. Set the matching API key as an environment variable:

| Provider | Environment variable | Notes |
|---|---|---|
| `anthropic` | `ANTHROPIC_OAUTH_TOKEN` or `ANTHROPIC_API_KEY` | |
| `openai` | `OPENAI_API_KEY` | |
| `gemini` | `GEMINI_API_KEY` | |
| `openrouter` | `OPENROUTER_API_KEY` | |
| `ollama` | *(none required)* | Needs a running Ollama instance |

To switch providers, edit `agent-bundle.yaml`:

```yaml
model:
  provider: ollama          # or openai, gemini, openrouter
  model: qwen2.5-coder      # model name for the chosen provider
```

## Example prompts

Try these in the WebUI chat:

- `Generate sample sales data for 12 months and show monthly revenue trends`
- `Create a dataset of 100 students with random grades, then show the grade distribution histogram`
- `Analyze the correlation between temperature and ice cream sales with sample data`

## What to expect

1. The agent writes a Python script to `/workspace/analysis.py`
2. Executes it in the E2B sandbox
3. Returns summary statistics and references generated files
4. The file tree panel shows generated files (`analysis.py`, `chart.png`, etc.)

## Features

- **WebUI**: Full chat interface with real-time streaming via WebSocket
- **TUI**: Terminal interface also available
- **File tree**: Browse `/workspace` files generated by the agent
- **E2B sandbox**: Secure cloud execution with pandas, matplotlib, numpy

## Success criteria

1. WebUI loads at the printed local URL (`http://localhost:3000` by default)
2. Agent generates and executes Python analysis scripts
3. Output includes statistical summaries and chart file references
4. File tree panel shows generated files under `/workspace`

## Cleanup

Stop server with `Ctrl+C`.
